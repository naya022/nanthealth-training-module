Escalation path — when something needs to move fast

How I'd handle urgent training or documentation gaps
1
Immediate Slack / Teams message to the relevant team lead
Fastest path — no ticket, no meeting. Flag the gap and propose a fix in the same message.
2
Same-day draft in Confluence if it's a documentation issue
A draft KB article or updated user guide goes up immediately — even if unpolished — so Support and CS have something to reference while a full fix is built.
3
Emergency Pendo tooltip if it's a product UX confusion issue
A one-sentence Pendo tooltip can go live in under an hour. For urgent confusion points surfaced by Support, this is the fastest training intervention available.
4
Follow-up email summary to all stakeholders once resolved
Short post-mortem: what the gap was, what was deployed, and what's being built long-term. Keeps everyone aligned without a meeting.
Section 05 — Technical tools in this role

How I'd use every tool named in the job description

The JD lists eight specific tools and platforms. Below is exactly how I'd use each one — mapped to real tasks in this role, not generic descriptions. Every tool is tied to a specific JD responsibility or success metric.

Tools named directly in the job description

Pendo
Digital adoption platform
"Leverage Pendo to design and deploy in-app guides, tooltips, and walkthroughs that drive user adoption"
  • Build and deploy provider onboarding tooltip sequences for NaviNet's highest-friction workflows (E&B, auth submission)
  • Use Pendo Analytics to track guide completion rates, step-level drop-off, and feature adoption delta by cohort
  • Create segmented guides — new providers vs. returning — so experienced users aren't shown beginner walkthroughs
  • Set up in-app announcements for product releases so providers learn about new features without leaving NaviNet
  • Share Pendo dashboards monthly with Customer Success as the primary adoption metric
Measures: guide completion %, feature adoption, support ticket delta
iSpring
e-learning authoring tool
"Proficiency with e-learning tools (e.g., iSpring, Vyond)"
  • Rebuild the prior authorization training module as a branching iSpring scenario — provider selects a diagnosis, submits an Eviti auth, receives rejection, corrects and resubmits
  • Convert existing PDF job aids into interactive iSpring modules with knowledge checks built in
  • Develop SCORM-compliant modules that upload directly into the LMS for completion tracking
  • Use iSpring's quiz builder to create post-training assessments tied to CSAT and knowledge retention goals
Measures: knowledge retention, module completion, CSAT improvement
Confluence
Knowledge management system
"Organize and optimize content within knowledge management systems (e.g., Confluence, SharePoint)"
  • Primary home for all customer-facing user guides, workflow documentation, and release notes
  • Build a live CSAT dashboard page that CS and Support can access anytime without waiting for a report
  • Maintain a documentation backlog page per Product release — every feature with a training deliverable tracked by status
  • Create and maintain a KB article tracker so Support knows exactly what articles are in draft, review, or published
  • Tag all content by product (NaviNet, Eviti Connect), audience (provider, payer, internal), and last-reviewed date for audit readiness
Measures: KB engagement, content freshness, audit readiness
SharePoint
Document management & intranet
"Organize and optimize content within knowledge management systems (e.g., Confluence, SharePoint)"
  • House version-controlled training materials, policy documents, and compliance records accessible to internal teams
  • Maintain a training library for Customer Success and Implementation with quick-reference guides and job aids
  • Use SharePoint metadata and search tagging so the right document is findable without knowing the exact filename
  • Archive superseded versions with clear change logs for HIPAA audit readiness
Measures: document retrieval time, version control compliance
DITA
Documentation standard
"Proficiency with documentation platforms (e.g., DITA)"
  • Structure user guides and release notes as topic-based, reusable content units — one source, published to multiple formats (HTML, PDF, in-app)
  • Apply DITA's task, concept, and reference topic types to NaviNet workflow documentation so content is modular and updateable without rewriting entire guides
  • Align to DITA standards ensures documentation is audit-ready and regulation-compliant for HIPAA review cycles

Vyond
Video & animation authoring
"Proficiency with e-learning tools (e.g., iSpring, Vyond)"
